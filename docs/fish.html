Fish Interface

Fish is designed for "out-of-the-box" brilliance. In this project, Fish has been enhanced with "Post-Execution" error capturing logic and 100% parity with the Bash command library.

Implementation Details

  • Abbrev System: Instead of standard aliases, Fish uses abbr which expands commands visually as you type.
  • Event Handlers: Uses --on-event fish_postexec to automatically log any command failure to errors.log.
⚠️ Note: Fish is not POSIX-compliant. For scripting, the project defaults to /bin/bash.

Universal Sourcing

Fish shares core logic from the core/ directory using a compatibility shim to ensure shared features like battery_warn work natively.
