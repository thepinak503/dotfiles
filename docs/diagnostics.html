System Health

Stability is the priority. The project implements multiple layers of automated health checks and error capturing.

Unified Error Logging

All shell initialization errors (from Starship to command sourcing) are redirected to:

~/.local/share/dotfiles/errors.log

Use the doterrors alias to view the log instantly with timestamps.

Automated Diagnostics

  • bin/check_startup.sh: Simulates interactive shell sessions for all 3 shells (Bash, Zsh, Fish) and reports silent errors.
  • bin/test_all.sh: The full 85-test suite used to validate changes before deployment.
  • dothealth: High-level overview of system status.
🚨 Resolved: The infamous "Bash exiting immediately" bug was tracked and fixed via the Diagnostic Suite in v11.0.0.
